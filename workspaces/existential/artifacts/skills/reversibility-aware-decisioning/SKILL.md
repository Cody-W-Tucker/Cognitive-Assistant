## When To Use
- The user asks for help deciding, committing, choosing a path, or reducing overthinking.
- The decision feels like a cliff but may be convertible into steps.
- The right response depends on reversibility, exposure, test criteria, or commitment rules.
- The user needs a way to preserve intuition while reducing unnecessary risk.

## Do Not Use
- For simple pros/cons where stakes are low and the user wants a quick recommendation.
- When the user has already chosen and only needs a plan.
- When the main issue is market translation or interpersonal signal triage.

## Capability Guidance
Avoid exhaustive abstract analysis as the default. Classify the decision by reversibility and design the smallest live test that can produce real information.

Decision flow:
1. Define the decision in one sentence.
2. Classify it:
   - Two-way door: reversible; run a small test quickly.
   - One-way door: hard to reverse; slow down, define thresholds, seek counsel/evidence.
   - Mixed door: reversible in parts; isolate the irreversible component.
3. Identify exposure: time, money, reputation, relationships, spiritual/moral cost, opportunity cost.
4. Define a test: what can be tried without pretending to decide forever?
5. Set criteria: what result would mean continue, pivot, stop, or commit?
6. Name the commitment threshold: what must be true before no-looking-back commitment is wise?

## Intervention Patterns
- Replace “What is the perfect answer?” with “What information do we need next?”
- Replace “Should I do this?” with “What is the smallest honest version of doing this?”
- For one-way doors, recommend deliberate commitment after thresholds are met, not endless hedging.
- End with a concrete next move within a short time window when possible.

## Failure This Prevents
Prevents generic pros/cons lists, false urgency, and analysis loops. It helps the user convert intuition and uncertainty into staged action without surrendering agency.
